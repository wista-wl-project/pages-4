# Release Policy（リリース・バージョン方針）

本ドキュメントは、  
WISTA プロジェクトにおける **バージョン番号の付け方**、  
**リリースの考え方**、および **互換性方針**を定義するものです。

---

## バージョニング方式

WISTA は、  
**Semantic Versioning（セマンティック バージョニング）**をベースにした  
独自の安定重視方針を採用します。
```
MAJOR.MINOR.PATCH
```
### 各番号の意味

#### MAJOR（破壊的変更）
- 既存の利用方法と **互換性がない変更**
- API / UI / 設定 / 動作仕様の大幅変更
- ライセンスや利用条件に影響する変更
- プロジェクト構造の大規模な再設計

👉 MAJOR バージョンアップは **慎重に行われます**

---

#### MINOR（機能追加）
- 後方互換性を保った新機能追加
- UI 改善・機能拡張
- 対応言語・再生方式の追加
- API の拡張（既存仕様を壊さないもの）

---

#### PATCH（修正・安定化）
- バグ修正
- セキュリティ修正
- 軽微な UI / 文言修正
- パフォーマンス改善

---

## リリースの基本方針

- **安定性を最優先**します
- 実験的機能はデフォルトで有効化しません
- 破壊的変更は事前告知を行います
- 無理な頻度でのリリースは行いません

「速さ」よりも  
**継続して安全に使えること**を重視します。

---

## プレリリースについて

以下のようなバージョンは **プレリリース**として扱います。
```
vX.Y.Z-alpha
vX.Y.Z-beta
vX.Y.Z-rc
```
### プレリリースの特徴

- 動作保証はありません
- 仕様が変更される可能性があります
- 本番利用は推奨されません

テスト・検証・フィードバック目的での利用を想定しています。

---

## Breaking Change（破壊的変更）の扱い

破壊的変更を含む場合、以下を行います。

- Release Notes に明記
- 変更内容・影響範囲の説明
- 移行方法の案内（可能な場合）

---

## リリースノート（Release Notes）

各リリースでは、以下の情報を記載します。

- 変更点の概要
- 追加された機能
- 修正された不具合
- 既知の問題（ある場合）
- Breaking Change の有無

GitHub Releases を正式なリリース情報とします。

---

## ライセンスとの関係

すべてのリリースは  
**Wista 非商用コピーレフト・事前通知義務ライセンス（WNCCL-PN）**  
のもとで提供されます。

バージョンアップによって  
商用利用が自動的に許可されることはありません。

---

## サポート方針

- 最新の安定版を主なサポート対象とします
- 古いバージョンへの修正対応は保証されません
- 重大なセキュリティ問題は例外的に対応する場合があります

---

## フィードバックについて

- バグ報告・改善提案は Issue からお願いします
- 破壊的変更の提案は事前に Discussion / Issue で相談してください

---

## 最後に

WISTA は、  
**非商用・自由・持続可能**であることを大切にしています。

このリリース方針は、  
プロジェクトと利用者双方を守るためのものです。

ご理解とご協力をお願いします。
